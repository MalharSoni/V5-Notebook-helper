## Design Philosophy

### Core Principles

1. **Content Over Style**
   - RECF explicitly states evaluations prioritize "content and clarity over sophistication of presentation"
   - Visual design should ENHANCE readability, not distract from content
   - A beautiful notebook with weak content will lose to a plain notebook with excellent content

2. **Consistency is Key**
   - Same visual language throughout entire notebook
   - Consistent colors, fonts, spacing, and layouts
   - Creates professional impression and aids navigation

3. **Form Follows Function**
   - Every design element should serve a purpose
   - Color coding helps judges navigate
   - Visual hierarchy guides reading order
   - White space prevents overwhelm

4. **Student-Created Aesthetic**
   - Should look like students made it (because they did)
   - Over-polished can appear adult-assisted
   - Authentic > Perfect

## Color Systems

### Team 839Z Current Color Scheme
Based on notebook analysis:
- **Primary**: Bright Yellow (#FFFF00) - Background, section dividers
- **Secondary**: Red (#FF0000) - VEX branding, accents
- **Tertiary**: Blue, Green, Purple, Orange - Tables, data, sections

### Recommended Color System

#### Primary Brand Colors
```
Yellow (Brand Primary):    #FFFF00 or #FFD700
Black (Text):              #000000 or #1A1A1A
White (Background):        #FFFFFF
```

#### Section Color Coding (EDP-Based)
| Section | Color | Hex Code | Meaning |
|---------|-------|----------|---------|
| Introduction/Team | Yellow | #FFD700 | Identity |
| Game Analysis | Purple | #8B5CF6 | Strategy |
| Mechanical Design | Blue | #3B82F6 | Engineering |
| Programming | Green | #22C55E | Code |
| Testing | Orange | #F97316 | Evaluation |
| Competition | Red | #EF4444 | Performance |
| Project Management | Gray | #6B7280 | Administration |

#### Color Accessibility
- Ensure sufficient contrast (4.5:1 minimum)
- Don't rely on color alone for information
- Test with color blindness simulators
- Use patterns/icons alongside colors

### Color Usage Guidelines
1. **Background**: Keep white or very light for readability
2. **Headers**: Can use brand colors
3. **Section Tabs**: Use designated section colors
4. **Text**: Black or very dark gray
5. **Accents**: Use sparingly for emphasis
6. **Tables**: Alternate row colors for readability

## Typography

### Font Selection

#### Recommended Font Pairings
**Option 1 (Clean & Modern)**
- Headlines: Arial Bold or Helvetica Bold
- Body: Arial or Calibri
- Code: Consolas or Courier New

**Option 2 (Professional)**
- Headlines: Montserrat Bold
- Body: Open Sans
- Code: Source Code Pro

**Option 3 (Engineering Feel)**
- Headlines: Oswald or Roboto Condensed
- Body: Roboto
- Code: Roboto Mono

### Font Sizes
| Element | Size Range |
|---------|------------|
| Page Title | 24-32pt |
| Section Headers | 18-24pt |
| Subsection Headers | 14-18pt |
| Body Text | 11-12pt |
| Captions | 9-10pt |
| Footnotes | 8-9pt |

### Typography Rules
1. **Maximum 2-3 fonts** in entire notebook
2. **Consistent sizing** throughout
3. **Clear hierarchy** - readers should know what's important
4. **Line spacing**: 1.15-1.5 for body text
5. **Paragraph spacing**: Add space between paragraphs
6. **Alignment**: Left-align body text (easier to read)

## Layout Principles

### Page Structure

```
+------------------------------------------+
|  [Header: Project | Title | Page #]      |
+------------------------------------------+
|     |                                    |
|  S  |                                    |
|  E  |     MAIN CONTENT AREA              |
|  C  |                                    |
|  T  |     - Text                         |
|  I  |     - Images                       |
|  O  |     - Tables                       |
|  N  |     - Diagrams                     |
|     |                                    |
|  T  |                                    |
|  A  |                                    |
|  B  |                                    |
+------------------------------------------+
|  [Footer: Designer | Witness | Date]     |
+------------------------------------------+
```

### Grid System
- Use consistent margins (0.5-0.75 inches)
- Align elements to invisible grid
- Consistent spacing between elements
- Balance visual weight on page

### White Space
- Don't fill every pixel
- White space improves readability
- Group related content together
- Separate distinct sections visually

### Visual Balance
- Heavy elements (images) balanced with lighter elements (text)
- Avoid clustering everything on one side
- Consider visual flow (Z-pattern or F-pattern reading)

## Visual Elements

### Headers and Footers

**Header Should Include:**
- Project/Robot name
- Page title
- Page number

**Footer Should Include:**
- Designer/Author name
- Witness name (if applicable)
- Date

**Example Header Layout:**
```
Project: Gen 1 Robot  |  Title: Drivetrain  |  Page: 45
```

### Section Divider Pages
- Full-page or half-page dividers between major sections
- Include section title prominently
- Use section color as background or accent
- Include decorative elements consistent with brand

### Tables
**Best Practices:**
- Clear headers (bold, different background)
- Alternating row colors for readability
- Consistent column widths
- Adequate cell padding
- Clear borders or separating lines

**Example Table Styling:**
```
+------------------+------------------+------------------+
| Header 1         | Header 2         | Header 3         |
| (Bold, Colored)  | (Bold, Colored)  | (Bold, Colored)  |
+------------------+------------------+------------------+
| Data Row 1       | Data             | Data             |
| (White BG)       |                  |                  |
+------------------+------------------+------------------+
| Data Row 2       | Data             | Data             |
| (Light Gray BG)  |                  |                  |
+------------------+------------------+------------------+
```

### Callout Boxes
Use for:
- Important notes
- Tips
- Warnings
- Key decisions

**Styling:**
- Distinct background color
- Border or shadow
- Icon to indicate type
- Clear contrast from body text

### Charts and Graphs
- Use when data visualization helps understanding
- Pie charts for composition
- Bar charts for comparisons
- Line charts for trends over time
- Always include titles and legends
- Use consistent color scheme

## Section Identification

### Margin Tabs
**Purpose**: Allow judges to quickly navigate to specific sections

**Implementation:**
- Colored bars in margin (consistent position)
- Can extend slightly beyond page edge
- Color corresponds to section type
- Can include abbreviated text label

**Team 839Z Current Implementation:**
Good use of vertical margin tabs with colors indicating:
- Yellow: General/Team
- Blue: Mechanical
- Green: Programming
- Purple: Strategy
- Orange: Testing

### Page Numbering
- Consistent location (top right or bottom center)
- Clear, readable size
- Can include total pages (e.g., "45 of 110")

### Running Headers
- Include section name in header
- Helps readers know where they are
- Useful for digital navigation

## Images and Diagrams

### Photo Quality
**Requirements:**
- Well-lit (avoid shadows)
- In focus
- Appropriate resolution (150+ DPI for print)
- Clean background when possible
- Show relevant details

### Photo Guidelines
1. **Label everything** - arrows, callouts, part names
2. **Multiple angles** when helpful
3. **Progress photos** - before/during/after
4. **Scale reference** when size matters
5. **Crop to relevant content**

### CAD Renderings
- Multiple views (isometric, top, front, side)
- Consistent background (white or subtle gradient)
- Parts labeled or exploded views
- Dimensions when relevant
- High resolution exports

### Diagrams
**Types to Include:**
- Field diagrams with dimensions
- Flow charts for processes
- Wiring diagrams
- Code flow diagrams
- Strategy diagrams

**Diagram Best Practices:**
- Use consistent shapes and colors
- Include legends
- Keep simple enough to understand quickly
- Label all elements

### Image Captions
- Every image should have a caption
- Explain what's shown and why it matters
- Keep concise but informative

## PowerPoint-Specific Tips

> Team 839Z edits their notebook in PowerPoint

### Slide Setup
- **Page Size**: 8.5" x 11" (Letter) or A4 for print
- **Orientation**: Portrait typically
- **Margins**: Guide lines at 0.5" from edges

### Master Slides
**Create masters for:**
- Title/Cover page
- Section dividers
- Standard content page
- Meeting minutes page
- Testing page
- Two-column layout
- Image-heavy layout

**Master slide elements:**
- Header and footer positions
- Section tabs
- Page number placeholder
- Consistent backgrounds

### Grid and Guides
- Enable View > Guides
- Set up consistent grid
- Use Align tools (Arrange > Align)
- Snap to grid for consistency

### Exporting
- **For Print**: PDF, High Quality
- **For Screen**: PDF, Standard
- **For Images**: PNG at 150+ DPI
- Check all fonts embed correctly
- Verify colors look correct

### Version Control
- Save with date in filename (e.g., Notebook_2025-01-15.pptx)
- Use OneDrive/Google Drive for collaboration
- Version history provides sequential documentation proof

### Performance Tips
- Compress images (Picture Format > Compress)
- Avoid excessive animations
- Keep file size manageable
- Use linked images for large photos

## Common Mistakes to Avoid

### Visual Mistakes
1. **Too many fonts** - Stick to 2-3 maximum
2. **Inconsistent colors** - Use defined color palette
3. **Crowded pages** - Leave white space
4. **Poor image quality** - Blurry or dark photos
5. **No visual hierarchy** - Everything same size/weight
6. **Distracting backgrounds** - Keep clean and simple
7. **Misaligned elements** - Use grid and guides
8. **Inconsistent spacing** - Standardize gaps

### Content Presentation Mistakes
1. **Walls of text** - Break up with images, bullets
2. **Unlabeled images** - Always caption and label
3. **Missing dates** - Every page needs date
4. **No page numbers** - Essential for navigation
5. **Unclear organization** - Use section tabs
6. **Missing attribution** - Always credit sources

### Technical Mistakes
1. **Font not embedding** - Check PDF export
2. **Colors shifting** - Use RGB for screen, CMYK for print
3. **Low resolution images** - 150+ DPI minimum
4. **File too large** - Compress and optimize
5. **Inconsistent page sizes** - Check all pages match

- [ ] All pages same size
- [ ] Page numbers sequential
- [ ] Fonts embedded
- [ ] Images high quality
- [ ] Colors consistent throughout
- [ ] No orphaned headers/elements
